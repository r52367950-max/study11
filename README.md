> 已根据你最新确认更新：高中 + 英语/数学/语文 + 语音先做 + 后台Memory化能力档案。

## 一、V1范围（已确认）

1. 目标学段：**高中**
2. 首发学科：**英语、数学、语文**
3. 语音功能：**优先做 AI 默写/听写**
4. 评估结果：**写入后台长期记忆（Memory）**，持续更新学生画像

---

## 二、核心功能拆解（按优先级）

## P0-1：AI解题与出题

### A. AI解题（英语/数学/语文）
输入：题目文本（后续扩展拍照OCR）
输出：
- 分步讲解
- 对应知识点
- 易错点
- 同类变式题（1~2道）

三科差异化策略：
- 数学：必须给推导步骤 + 常见误区
- 英语：语法点 + 词汇替换 + 例句
- 语文：考点定位（字词/病句/阅读主旨/文言）+ 答题模板

### B. AI出题
输入：学科、知识点、难度、题量
输出：题目 + 答案 + 解析 + 分值建议

V1建议：
- 数学：选择/填空/简答
- 英语：词汇、语法、阅读小题
- 语文：基础知识+阅读题（先规则评分）

---

## P0-2：AI默写/听写（语音模型）

### 用户闭环
1. 选择范围（词库/古诗文/课文）
2. 系统 TTS 播报
3. 学生语音作答（或文本输入）
4. ASR识别 + 标准答案对齐
5. 输出错词/漏词/近音错误与复习建议

### V1优先顺序
1. 英语单词听写（稳定、易量化）
2. 英语句子听写
3. 语文古诗文默写

### 评分维度
- 正确率
- 完整度
- 响应时长
- 连续稳定性（最近7天）

---

## P0-3：学生能力评估报告（Memory化）

## 核心思路
不是一次性报告，而是“**持续更新的后台能力档案**”（类似AI应用的 Memory）。

### 后台Memory记录内容
- 学生基础信息（学段、目标分数、年级）
- 学科能力向量（英语/数学/语文分维度）
- 知识点掌握度
- 错题偏好与易错模式
- 学习节奏（活跃时间、专注时长、拖延点）
- 建议执行记录（是否完成）

### 报告输出（前端可视化）
- 最新能力等级（A/B/C）
- 近7/30天趋势
- 本周3个优先提升点
- 个性化学习建议（可执行到天）

### 评分机制
V1：规则评分（正确率 + 难度 + 用时）
V2：能力估计模型（IRT/Rasch）

---

## P0-4：API接入能力

### 支持配置
- Provider（OpenAI兼容）
- Base URL
- API Key
- Model
- 超时/重试

### 安全策略
- Key仅服务端保存（加密）
- 前端不直连第三方模型
- 请求日志脱敏

### 成本策略
- 题目生成缓存
- 分级模型路由（简单任务小模型，复杂任务升级）
- 限频、配额与熔断

---

## 三、建议系统架构（V1）

- `Tutor Service`：解题、出题、讲解
- `Speech Service`：TTS/ASR/对齐评分
- `Assessment Service`：评分与报告生成
- `Memory Service`：学生长期画像与建议追踪
- `Gateway/API Config`：多模型接入与鉴权

数据层：
- 用户与配置表
- 题目与作答记录表
- 音频任务记录表
- 学生能力档案表（Memory）
